Endpoints

Simple routes, useful payloads

The API is intentionally small and predictable so developers can use it for demos, integration tests, dashboards, API tooling, and QA workflows.

GET /api/v1/health

Check service status, version, and uptime.

GET /api/v1/gadgets?limit=4&sort=rating_desc

List gadgets with pagination, sorting, category, brand, and price filters.

GET /api/v1/gadgets/104

Fetch a single record by numeric id.

GET /api/v1/gadgets/slug/nova-x1

Fetch a single record by stable slug.

GET /api/v1/search?name=drift

Search records by partial name, brand, category, or tags.

GET /api/v1/random

Get a random gadget for demo flows and UI loading states.

GET /api/v1/categories

Return category names and counts to drive menus or filters.

GET /api/v1/brands

Return brand names and counts for filtering and docs.
